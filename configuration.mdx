## Setting Brand Colors and Themes

Customize your space's appearance to align with your brand. Update the primary color, logo, and theme mode.

<Steps>
  <Step title="Access Settings" icon="settings">
    Navigate to `https://dashboard.example.com/settings/branding`.
  </Step>
  <Step title="Update Brand Color" icon="palette">
    Enter your hex code, such as `#3B82F6` for a blue theme.
  </Step>
  <Step title="Choose Theme" icon="moon">
    Select light, dark, or auto mode.
  </Step>
  <Step title="Preview and Save" icon="check-circle">
    Preview changes and click Save.
  </Step>
</Steps>

```json
{
  "brandColor": "#3B82F6",
  "theme": "auto",
  "logoUrl": "https://your-logo.example.com/logo.png"
}
```

<Callout kind="info">
  Brand color affects buttons, links, and accents. Use tools like coolors.co to generate palettes.
</Callout>

## Managing User Permissions and Roles

Control who can view, edit, or admin your documentation.

| Role       | View | Edit | Delete | Invite Users |
|------------|------|------|--------|--------------|
| Viewer     | ✅   | ❌   | ❌     | ❌           |
| Editor     | ✅   | ✅   | ❌     | ❌           |
| Admin      | ✅   | ✅   | ✅     | ✅           |

<Tabs>
  <Tab title="Add User" icon="user-plus">
    1. Go to `https://dashboard.example.com/settings/users`.
    2. Enter email and select role.
    3. Send invite.
  </Tab>
  <Tab title="Role Permissions" icon="shield">
    Customize per project:
    
````json
{
  "projectId": "proj_123",
  "roles": {
    "editor": ["read", "write"],
    "admin": ["read", "write", "delete", "manage_users"]
  }
}
````
  </Tab>
</Tabs>

<Expandable title="Advanced Role Customization" default-open="false">
  Use API to set granular permissions:
  
  <Request tabs="cURL,JavaScript">
  ```
  curl -X POST https://api.example.com/v1/permissions \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -d '{"userId": "user_456", "permissions": ["edit_pages"]}'
  ```
  ```javascript
  const response = await fetch('https://api.example.com/v1/permissions', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      userId: 'user_456',
      permissions: ['edit_pages']
    })
  });
  ```
  </Request>
</Expandable>

## Configuring Integrations

Connect * to tools like GitHub, Slack, or webhooks.

<CodeGroup tabs="GitHub,Webhook">
```json
{
  "integration": "github",
  "repo": "your-org/your-repo",
  "webhookSecret": "YOUR_SECRET"
}
```
```json
{
  "integration": "webhook",
  "url": "https://your-webhook-url.com/webhook",
  "events": ["page_updated", "user_invited"]
}
```
</CodeGroup>

<ParamField body="integration" param-type="string" required="true">
  Type of integration: `github`, `slack`, `webhook`.
</ParamField>

<ParamField body="config.repo" param-type="string">
  Repository name for GitHub sync.
</ParamField>

## Exporting and Backups

Protect your content with regular exports.

<Steps>
  <Step title="Manual Export" icon="download">
    Visit `https://dashboard.example.com/settings/export` and select Markdown or PDF.
  </Step>
  <Step title="Schedule Backup" icon="calendar">
    Set frequency: daily, weekly.
  </Step>
  <Step title="Download" icon="save">
    Retrieve ZIP from your storage.
  </Step>
</Steps>

<Callout kind="alert">
  Enable automatic backups to avoid data loss. Store exports securely off-site.
</Callout>
